# AGENTS.md — Правила работы с циклами разработки

> Переносимый набор правил для работы в Blackbox, Cursor, Continue и других средах (оптимизировано под сильные модели текущего поколения).

## Основной цикл (Solver Loop)

1. **Inspect** — сначала полностью прочитай релевантный код, логи, состояние БД, текущие тесты. Не начинай править, пока не поймёшь «spine» системы.
2. **Define success** — чётко опиши measurable исход (тест прошёл, smoke-эндпоинт вернул ожидаемый JSON, эскалация создана с правильным reason и т.д.).
3. **Smallest vertical slice** — реализуй минимальный end-to-end кусок, который доказывает, что основная идея работает.
4. **Proportional verification** — проверь на поверхности пользователя (тесты + ручная симуляция + логи). Фиксируй доказательства.
5. **Reflect** — что сработало, что нет, нужно ли откатить через git. Только после этого расширяй scope.

Максимум 3 инструмента/вызова без рефлексии внутри роли.

## Два режима работы

- **Agentless (рекомендуется для большинства задач)**: одна сильная модель следует Solver Loop напрямую. Лёгкий оверхед, быстрые итерации. Поддержка в `agentless_loop/`. Используй по умолчанию в этом CLI и Cursor.
- **Agentic (полноценный)**: многоролевая машина (Orchestrator → Coder → Tester → Reviewer). Больше проверок, больше артефактов. Шаблон в `agentic_loop_template/`. Подключай при высокой сложности или когда нужно жёсткое разделение ответственности.

## Важные технические рекомендации

- При работе с моделями, поддерживающими reasoning traces, сохраняй их в истории контекста — это значительно повышает качество длинных итераций.
- Для творческих ролей (реализация, отладка) полезны более высокие значения temperature (0.7–1.0). Для планирования, тестирования и ревью — низкие (0.0–0.2).
- Хорошие модели текущего поколения отлично справляются с tool calling и структурированным выводом — активно используй эти возможности.
- Контекст больших моделей позволяет работать с крупными репозиториями. Агрессивно суммируй историю, оставляй только релевантные факты, метрики прошлых попыток и ключевые решения.

## Постоянные правила проекта (kurbanov)

- Все коммиты, комментарии в коде и докстринги — только на естественном русском, стиль реального mid/senior разработчика.
- **Полный запрет** упоминания AI, LLM, агент, нейросеть, MiniMax, M2.7, Grok, Claude и т.п. в коммитах и комментариях к коду.
- Любая работа с Python — только внутри `.venv` после `setup_kurbanov.ps1`.
- Perplexity (sonar-pro) в этом проекте используется **исключительно** для генерации текстов ответов консультанта. Вся логика квалификации, эскалаций и калькулятора — строго детерминированная.
- Бот никогда не раскрывает клиенту факт использования внешних сервисов.
