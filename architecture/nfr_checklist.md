# NFR checklist — MVP Поток 1 (baseline)

## Надежность
- [ ] Повторная обработка webhook не приводит к потере данных (idempotency strategy)
- [ ] Ошибки Telegram API не ломают прием входящих сообщений (ретраи/логирование)

## Безопасность
- [ ] Включен `TELEGRAM_WEBHOOK_SECRET_TOKEN`
- [ ] Для Admin API задан `ADMIN_API_TOKEN`
- [ ] Ролевые права закреплены и проверяются на уровне API (RBAC)

## Наблюдаемость
- [ ] Записываются события: диалог, эскалация, расчет, действия менеджера (BOT-UAT-09)
- [ ] Есть минимальные метрики: число диалогов, эскалаций, доля вне контура v1

## Производительность
- [ ] Поддержка до 10 одновременных диалогов (baseline из `pre_kp_customer_questionnaire_and_input_data.md`)
- [ ] Таймауты внешних запросов (CBR, Telegram) ограничены
