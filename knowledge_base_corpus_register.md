# Реестр корпуса базы знаний

**Проект:** Telegram-консультант по подбору автомобилей  
**Версия:** 1.0  
**Статус:** baseline структуры зафиксирован для MVP; наполнение контентом выполняет заказчик  
**Дата актуализации:** 14.05.2026  
**Связанный реестр решений:** `decision_log.md`

## 1. Назначение

Реестр задает рабочую структуру базы знаний для MVP и последующего развития. Целевой объем корпуса 5-10 млн символов рассматривается как рамка развития, а не как обязательный объем первого релиза без отдельного подтверждения scope и бюджета.

Правило качества: если ответа нет в подтвержденной базе, источник устарел, данные спорные или уверенность низкая, бот передает обращение менеджеру.

Допустимые статусы по разделам корпуса:
- `open` - решение не принято.
- `waiting_customer` - требуется подтверждение или входные данные заказчика.
- `resolved` - зафиксировано для MVP и может использоваться в точном КП.
- `deferred` - вынесено за рамки MVP (не входит в точное КП MVP).

## 4. Статусы модерации

| Статус | Значение |
|---|---|
| `waiting_customer` | Материал или разрешение на использование ожидается |
| `draft` | Материал загружен, но не проверен |
| `approved` | Материал проверен владельцем и может использоваться в ответах |
| `restricted` | Материал можно использовать только как подсказку менеджеру |
| `deprecated` | Материал устарел и исключается из ответов |

## 5. Правила использования в ответах

1. Ответы по стоимости должны ссылаться на версию формулы, курс и дату расчета.
2. Ответы по комплектациям должны иметь источник и уровень уверенности.
3. При конфликте источников бот не выбирает версию самостоятельно, а передает вопрос менеджеру.
4. Непроверенные материалы не используются для финальных коммерческих ответов.
5. Обновление корпуса фиксируется датой, владельцем и статусом модерации.
6. В MVP запрещены автоматический парсинг и копирование контента сторонних сайтов: допускаются только ссылки и справочные метаданные, а также материалы, переданные заказчиком.
