# Официальный документ: Требования к проекту (Requirements Specification)

## Область
- Продукт A: Telegram-бот подбора автомобиля.
- Продукт B: Агрегатор логистических предложений.

## Цели
- Формализовать функциональные и нефункциональные требования.
- Зафиксировать требования к входным данным, интеграциям, ролям, приемке.

## Ключевые требования
1. Управляемый диалоговый контур с передачей обращения менеджеру.
2. Использование базы знаний с контролем источников данных.
3. Расчетный модуль итоговой стоимости по утвержденным формулам.
4. Интеграционный контур логистических источников с единой моделью данных.
5. Формальные критерии приемки и UAT по критичным бизнес-сценариям.
