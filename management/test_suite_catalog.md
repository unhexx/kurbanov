# Test suite catalog — MVP Поток 1

**Назначение:** зафиксировать состав Smoke/Regression/UAT наборов и правила их использования как quality gates.

**Источники правды:**
- UAT baseline: `uat_acceptance_scenarios.md`
- Gate критерии: `management/go_no_go_checklist.md`
- Трассируемость: `management/traceability_matrix.md`, `management/qa_traceability_matrix.md`

**Дата актуализации:** 14.05.2026

## 1) Smoke suite (на каждый деплой)
**Цель:** быстро подтвердить, что критичный пользовательский контур жив и не сломан.

**Триггер запуска:** каждый деплой/горячая правка, перед демонстрацией, перед UAT с заказчиком.

**Состав (UAT IDs):**
- BOT-UAT-01 (первичная квалификация)
- BOT-UAT-03 (расчет под ключ) — если ставки/курсы/формула в baseline готовы
- BOT-UAT-04 (нестандартный расчет → эскалация)
- BOT-UAT-07 (неизвестный вопрос → эскалация)
- BOT-UAT-08 (takeover менеджером + `/resume_bot`)
- BOT-UAT-09 (журналирование)

**Доп. проверки (не UAT ID, но обязательны для smoke):**
- Admin API health: `GET /admin/health` (см. `architecture/api_contracts.md`)

## 2) Regression suite (перед релизом/мерджем значимых изменений)
**Цель:** поймать регрессии по полному MVP потока 1.

**Триггер запуска:** перед релизом, после крупных изменений диалога/KB/калькулятора/аудита, после миграций БД.

**Состав (UAT IDs):**
- BOT-UAT-01..09 (все сценарии Telegram-консультанта)
- CALC-UAT-01..07 (калькулятор) — при наличии определений в `uat_acceptance_scenarios.md`

**Правило GAP:** если `uat_acceptance_scenarios.md` не содержит `CALC-UAT-*`, набор помечается как `blocked` до фиксации UAT baseline.

## 3) UAT suite (заказчик/ответственные подписанты)
**Цель:** приемка MVP по согласованным сценариям.

**Триггер запуска:** финальная приемка перед релизом.

**Состав:**
- BOT-UAT-01..09 (как минимум для MVP Поток 1)
- CALC-UAT-01..07 — если калькулятор выносится в отдельные UAT сценарии и они зафиксированы в baseline

**Артефакт выполнения (обязательный):**
- заполненный отчет по шаблону `management/uat_execution_report_template.md` (как отдельный файл вида `management/uat_execution_report_YYYY-MM-DD.md`).
