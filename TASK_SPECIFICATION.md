## 1. Общая цель проекта (MVP)

Создать и поддерживать работоспособный MVP сервиса **Telegram-консультанта по автоподбору автомобилей** для Exception.Expert со следующими компонентами:

- Telegram-бот (`Автоподбор - Exception.Expert`) с идемпотентной обработкой обновлений через webhook.
- Публичный веб-портал с переходом к анкете подбора и калькулятору.
- Анкета `/intake` — сбор данных клиента → создание лида в БД.
- Калькулятор итоговой стоимости `/calculator` (формула v1.0 с ограничениями).
- Веб-админка `/admin` для просмотра лидов, управления ставками, курсами ЦБ, базой знаний и пользователями.
- JSON API администрирования под `/admin/*`.
- Интеграция с внешним сервисом Perplexity для генерации ответов консультанта в диалоге.

**Текущий фокус (feature/telegram-ai-consultant-perplexity):**  
Завершить и стабилизировать интеграцию Perplexity (sonar-pro), улучшить качество ответов, обработку ошибок/эскалаций, обновить пользовательскую и операторскую документацию.

## 3. Жёсткие ограничения и правила (не нарушать никогда)

- **Детерминизм ядра**: Вся логика квалификации клиента, расчёта стоимости (калькулятор), правил передачи менеджеру — **строго детерминированная**. Обращения к Perplexity разрешены **только** для генерации текстов ответов консультанта.
- **Не раскрывать ИИ**: Бот ни при каких обстоятельствах не сообщает клиенту, что общается с искусственным интеллектом или использует внешние сервисы. Все ответы должны звучать как работа живого опытного менеджера Exception.Expert.
- **Язык артефактов разработки**:
  - Git commit messages — только естественный русский, стиль реального mid/senior разработчика, работавшего над проектом несколько месяцев.
  - Комментарии в коде, docstrings, логи — русский язык.
  - **Полный запрет** на слова AI, LLM, агент, нейросеть, MiniMax, Grok, Claude, "as an assistant" и т.п. в коммитах и комментариях к коду.
- **Локальная разработка**: По умолчанию использовать smoke-режим на SQLite (без Docker и PostgreSQL) для быстрых итераций. Полноценный режим — через `docker compose`.
- **Переменные окружения**: Чувствительные ключи (TELEGRAM_BOT_TOKEN, PERPLEXITY_API_KEY, ADMIN_API_TOKEN и др.) — только в `.env` (никогда не коммитить). При пустом `PERPLEXITY_API_KEY` консультант обязан gracefully переходить в rule-based режим (задаёт вопросы анкеты без модели).
- **Формула калькулятора**: Ограничения v1 — РФ, 3–5 лет, ≤160 л.с. Версия формулы через `FORMULA_VERSION`.
