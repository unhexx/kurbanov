# Предварительный опросник для подготовки Коммерческого предложения (КП)

**Проект:** Разработка Telegram-бота для квалификации клиентов по покупке автомобилей + Агрегатор логистики
**Дата:** 10.05.2026
**Исполнитель:** 

## 1. Фиксация входных данных (просим подтвердить)

Из расшифровки Voice-to-Text зафиксировано следующее:

**Основной проект — Telegram-бот:**
- Полностью conversational (natural language).
- Автоматическая квалификация клиента: марка, модель, бюджет (главный параметр), цвет, пробег и т.д.
- Фильтрация по бюджету.
- RAG по большой базе знаний (5–10 млн символов) — цены, характеристики авто.
- Адаптация цен под страны (Россия, Беларусь, Казахстан).
- Handover менеджеру: бот **не должен** отвечать, когда менеджер уже ведёт диалог.
- Калькулятор полной стоимости (растаможка, логистика и т.д.) — только по России на первом этапе.
- Парсинг данных с сайтов поиска авто.
- Отправка ссылок и видео-инструкций.
- Масштаб: ~10 одновременных клиентов.

**Второй проект (агрегатор логистики):** 
- Агрегация объявлений с 10–12 платформ.
- Заказчик предоставляет API-ключи.
- Единый дашборд с мощными фильтрами.

Просим подтвердить корректность понимания вышеуказанного или внести корректировки.

## 2. Краткий опросник (недостающие данные)

### Общие вопросы
1. Подтверждаете ли зафиксированные выше требования?
2. Какой приоритет проектов: сначала Telegram-бот (MVP) или оба проекта параллельно?

4. Кто с вашей стороны будет принимать работу (технический специалист/менеджер)?

### По Telegram-боту
5. В каком формате будет предоставлена база знаний (PDF, Excel, TXT, JSON)?
6. Как часто потребуется обновлять базу знаний?

8. Как именно должен работать handover (описание процесса)? Сколько менеджеров?
9. Список сайтов для парсинга цен и поиска автомобилей.

### По калькулятору

11. Есть ли готовые таблицы/формулы по логистике и растаможке?

### По агрегатору логистики

13. Для каких платформ есть API, для каких — только парсинг?
14. Какие фильтры и функционал критичны в первую очередь?

### Технические и организационные
15. Предпочтительный способ хостинга (свой сервер / облако)?
16. Нужны ли интеграции с CRM, 1С или другими системами?
17. Формат оплаты (аванс + этапы)?
