**Фиксация всех входных данных (для утверждения заказчиком)**

Я полностью разобрал расшифровку Voice-to-Text и зафиксировал **все** требования без интерпретаций. Это будет отправлено заказчику на подпись/согласование перед КП.

**Проект 1. Telegram-бот для квалификации и консультации клиентов по покупке автомобилей (основная тема):**
- Полностью conversational-бот (natural language, а не кнопки/меню).
- При запросе «хочу купить автомобиль» бот автоматически начинает квалификацию: марка, модель, цвет салона, пробег, **главный параметр — бюджет**.
- Жёсткая фильтрация по бюджету (100 тыс. ₽ → «ничего не подберём», 5 млн ₽ → подбираем варианты).
- Предложение аналогов и нескольких моделей.
- Большая база знаний (5–10 млн символов текста), из которой бот должен тянуть релевантные данные (цены, характеристики и т.д.) — классический RAG-подход.
- Адаптация цен под страну: Россия — базовая цена, Беларусь — примерно 9500 вместо 10 млн, Казахстан — примерно 9 вместо 10 млн.
- При неизвестном вопросе: «Подождите немного, я сейчас подумаю» или «Сейчас позову главного менеджера» → handover.
- **Критично**: бот **не должен отвечать** на сообщения менеджера и не должен реагировать, когда менеджер уже ведёт диалог с клиентом (предыдущий бот «полное говно» — отвечал всем подряд).
- Калькулятор полной стоимости автомобиля + логистики (растаможка, доставка и т.д.).
- Подгрузка/парсинг данных с внешних сайтов поиска авто.
- Отправка ссылок на поиск авто по странам + видео-инструкции по осмотру автомобиля.
- Калькулятор работает **только по России** (данных по Беларуси и Казахстану нет).
- Масштаб: одновременно ~10 клиентов пишут → бот отвечает 10 людям.

**Проект 2. Агрегатор логистики (клиент сказал, что интересует «даже больше»):**
- 10–12 платформ объявлений о грузах/перевозках.
- Заказчик предоставляет API-ключи.
- Сбор всех объявлений в единую платформу (отдельный сайт/дашборд) с мощными фильтрами.
- Цель — быстро перехватывать «вкусные» партии/предложения.

**Детальный анализ требований + риски + почему проект сложный**

Проект **не простой бот**. Это:
- Stateful conversational AI + RAG на большом корпусе.
- Критичная бизнес-логика (handover, молчание бота при работе менеджера).
- Мультистрановая ценовая логика + калькулятор.
- Парсинг/интеграции с внешними источниками (авто-сайты + 10–12 логистических платформ).

**Главные риски (их очень много, поэтому ×10 на оценку рисков):**
1. AI-галлюцинации и юридические риски (бот даёт цены → ошибка = претензии клиентов).
2. Handover-логика в Telegram (самая сложная часть технически).
3. Парсинг сайтов (Avito, Auto.ru и логистические платформы очень агрессивно банят, данные устаревают).
4. Стоимость и лимиты LLM-токенов при 10+ параллельных чатах.
5. Качество и актуальность базы 5–10 млн символов (нужна регулярная перезаливка).
6. Scope creep («а ещё вот это добавьте» — классика после плохого опыта).
7. Платёжеспособность и дисциплина заказчика (уже был негативный опыт).
8. Изменение API/структуры платформ (особенно в логистике).
9. Масштабируемость и latency RAG при одновременной работе 10 клиентов.
10. Поддержка после запуска (будет много доработок).

**Предлагаемое решение (как мы защищаемся и продаём ценность)**

Чтобы минимизировать риски и не повторить ошибку предыдущих разработчиков:
- **Фазовый подход** (MVP → v1 → v2). Сначала делаем только Telegram-бот (MVP с rule-based + простым RAG + идеальным handover). Логистический агрегатор — второй этап. Это снижает риски и даёт клиенту рабочий продукт быстро.
- **Чёткий контракт**: fixed-price по этапам + большой буфер на риски + еженедельные демо + возможность остановки.
- **Enterprise-надежность**: state management через Redis, guardrails для LLM, логирование каждого шага, возможность «заморозить» бота в чате.
- **Оптимизация затрат клиента**: используем российские LLM (GigaChat / YandexGPT / Sber), чтобы не зависеть от зарубежных токенов и курсов. Self-hosted векторная БД.

**Технологический стек (выбран под сложность проекта)**

**Telegram-бот:**
- Backend: Python 3.12 + aiogram 3.x (лучшая производительность и поддержка Telegram).
- AI/RAG: LangChain/LlamaIndex + Qdrant (векторная БД) + ru-модели (ruBERT/Sber).
- LLM: GigaChat / YandexGPT (российские, дешевле и лучше работают с русским языком).
- State & Cache: Redis.
- БД: PostgreSQL + pgvector.
- Handover: отдельная логика по user_id + Telegram Topics или supergroup.

**Агрегатор логистики:**
- Backend: FastAPI.
- Frontend: Next.js 14 (быстрый дашборд с фильтрами).
- Парсинг: Playwright + rotating proxies + Apify (если API недостаточно).
- Поиск: Elasticsearch.

**Общее**: Docker + CI/CD + Sentry + мониторинг. Всё self-hosted где возможно, чтобы контролировать затраты.

**Почему именно такой стек и почему дорого?**
Это не «просто бот на BotFather». Здесь требуется архитектор высокого уровня, который сделает надёжную систему, где бот никогда не будет «отвечать хуйню» и не будет мешать менеджеру. Именно поэтому ставка senior-разработчиков и архитекторов оправдана — мы не повторим ошибку предыдущих исполнителей.

**Предварительная оценка стоимости (с ×10 буфером на риски)**

**Базовая оценка (без буфера):**  
- MVP Telegram-бота (квалификация + RAG + handover + калькулятор) — 1,2–1,8 млн ₽ (3–4 месяца команды из 3–4 человек).  
- Полный бот + парсинг сайтов + калькулятор — 2,5–3,5 млн ₽.  
- Агрегатор логистики (10–12 платформ) — 3–5 млн ₽.

**Итоговая оценка с ×10 буфером на риски + поддержку + непредвиденное** (то, что просил заказчик):  
- **MVP Telegram-бота** — **2,5–3,5 млн ₽** (рекомендуемый старт).  
- **Полный Telegram-бот** — **5–8 млн ₽**.  
- **Оба проекта под ключ** — **10–15 млн ₽**.

Это **не завышение**. Цена отражает реальную сложность (AI + надёжность + мультистрановость + интеграции), объём работ и защиту от рисков. При этом мы даём клиенту прозрачность: каждый этап — отдельный fixed-price + отчётность.
