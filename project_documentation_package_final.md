## 2. Принцип объединения без потерь

Итоговая версия пакета объединяет материалы без исключения ранее согласованных блоков:
- формальная коммуникация с заказчиком и фиксация порядка согласования;
- краткий опросник для ускоренного сбора исходных данных перед КП;
- полный опросник для финальной детализации требований;
- проектный план с декомпозицией по этапам/эпикам/артефактам;
- черновики основных технических документов (Vision & Scope, SRS, Architecture, Roadmap);
- отдельный внутренний контур реестра рисков.

## 3. Роль краткого и полного опросников

1. **Краткий опросник** применяется первым и предназначен для:
   - уточнения текущего процесса заказчика;
   - фиксации целей развития;
   - первичного выбора технологического стека;
   - сокращения объема последующего согласования.

2. **Полный опросник** заполняется после краткого и интервью, непосредственно перед финальной подготовкой КП.

## 4. Условия перехода к финальному КП

Переход к выпуску точного финального КП выполняется после:
- заполнения краткого опросника;
- закрытия перечня недостающих входных данных;
- заполнения полного опросника;
- подтверждения требований к MVP, интеграциям, SLA, приемке и завершению этапов.

До закрытия этих условий допускается только диапазонная оценка с декомпозицией по волнам реализации.
