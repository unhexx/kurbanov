# Официальный документ: Техническое задание (ТЗ)

## 1. Основание
ТЗ формируется на основании согласованных требований и подтвержденных входных данных заказчика.

## 2. Состав работ
- Поток 1: проектирование и реализация MVP Продукта A — Telegram-консультанта по подбору автомобиля.
- Поток 2: подготовка и реализация первой очереди Продукта B — логистического агрегатора — после запуска и стабилизации MVP Продукта A.
- Интеграции, тестирование, приемка и эксплуатационные регламенты выполняются по каждому потоку отдельно.

## 2.1 Поток 1 — MVP Telegram-консультанта
- Управляемый диалог с клиентом и сбор обязательных параметров подбора.
- База знаний, расчетный модуль и правила передачи обращения менеджеру.
- Административные настройки, журналирование и приемка по сценариям Telegram-консультанта.
- Расчетный модуль реализуется только после утверждения `calculator_formula_specification.md`.
- База знаний используется только в рамках подтвержденных источников из `knowledge_base_corpus_register.md`.

## 2.2 Поток 2 — первая очередь логистического агрегатора
- Подготовка API-матрицы по 15 логистическим площадкам с фиксацией доступов, ограничений, лимитов и допустимых способов интеграции.
- Выбор 2–4 источников первой очереди для запуска агрегатора.
- Реализация единой модели данных, витрины предложений, фильтров v1, истории поиска, избранного и быстрого возврата к сохраненному фильтру.
- Подготовка уведомлений в согласованные каналы и приемка с участием заказчика и Тамары.
- Подключение источников выполняется только после подтверждения `logistics_api_matrix.md`.

## 3. Результаты
- По Потоку 1: реализованный и принятый MVP Telegram-консультанта в согласованном составе.
- По Потоку 2: реализованная и принятая первая очередь логистического агрегатора после подтверждения условий запуска.
- Акты приемки по утвержденным acceptance criteria для каждого потока.
- Комплект эксплуатационной документации по реализованным контурам.
- Актуальный `decision_log.md` со статусом всех решений, влияющих на точное КП.

## 4. Критерии завершения проекта
1. Закрыты все обязательные требования, вошедшие в scope.
2. Пройдены UAT-сценарии и подписаны результаты приемки.
3. Зафиксированы эксплуатационные процедуры и ответственность сторон.
4. Для Потока 2 дополнительно подтверждены API-матрица, источники первой очереди, обязательные поля карточки груза, фильтры v1 и отсутствие критических дефектов Severity 1/2.
